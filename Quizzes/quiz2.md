<font size="+2">2. </font> When writing a document in R Markdown, how do you denote the beginning of an R code chunk?

<hr>
<font size="+1"> <b>

-   \`\`\`{𝚛}

</b> </font>

<hr>
<hr>
<font size="+2">3. </font> When using knitr, how do you indicate the height and width of a plot created in a code chunk?

<hr>
<font size="+1"> <b>

-   Set the 'fig.height' and 'fig.width' options for the code chunk

</b> </font>

<hr>
<hr>
<font size="+2">4. </font> With some code chunks, we may not want the output generated by the chunk to be rendered into HTML but would prefer to print the output verbatim. How can we specify this preference for a given code chunk?

<hr>
<font size="+1"> <b>

-   𝚛𝚎𝚜𝚞𝚕𝚝𝚜 = "𝚊𝚜𝚒𝚜"

</b> </font>

<hr>
<hr>
<font size="+2">5. </font> When using knitr and R Markdown and producing output in HTML, why should you never edit the resulting HTML file?

<hr>
<font size="+1"> <b>

-   Every time you knit() the R Markdown file, the HTML file will be overwritten

</b> </font>
